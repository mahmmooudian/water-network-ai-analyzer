# Experimental Results

## Overview

This directory is intended to organize selected analytical outputs generated by the Water Network AI Analyzer.

The application provides evaluation and visualization workflows for machine learning-based pressure prediction and Particle Swarm Optimization-based PRV analysis.

Only selected, non-sensitive, and reproducible outputs should be included in the public repository.

---

## Result Categories

The project generates analytical outputs across two primary workflows:

1. Machine Learning-Based Pressure Prediction
2. PSO-Based PRV Optimization

Additional visualization and exported data outputs may also be generated during interactive analysis.

---

## Machine Learning Results

The XGBoost modeling workflow evaluates predictive performance using regression metrics.

Supported evaluation metrics include:

| Metric | Description                                                                    |
| ------ | ------------------------------------------------------------------------------ |
| MAE    | Mean Absolute Error between observed and predicted values                      |
| RMSE   | Root Mean Squared Error with increased sensitivity to larger prediction errors |
| R²     | Proportion of target variance explained by the regression model                |
| MAPE   | Percentage-based prediction error used in selected prediction workflows        |

The application also records model training time and the best hyperparameter configuration identified during randomized hyperparameter search.

## Prediction Visualizations

The application can generate actual-versus-predicted scatter plots for regression analysis.

These figures are intended to provide a visual comparison between observed network measurements and model predictions.

A regression trend and perfect-prediction reference line are included in the visualization workflow.

Example output category:

```text
figures/
└── actual_vs_predicted_pressure.png
```

## Feature Importance

When feature importance information is available from the trained estimator, the application can generate a feature importance visualization.

This output can support exploratory analysis of the relative contribution of model input variables.

Example output category:

```text
figures/
└── feature_importance.png
```

Feature importance should be interpreted as a model-specific analytical indicator and not as evidence of physical causality within the hydraulic network.

## PSO Optimization Results

The Particle Swarm Optimization workflow evaluates candidate PRV configurations across the configured operating horizon.

The application reports optimization indicators including:

| Indicator                   | Description                                                              |
| --------------------------- | ------------------------------------------------------------------------ |
| Optimization Penalty        | Mean objective function penalty                                          |
| Average Pressure            | Mean estimated pressure across optimized periods                         |
| Pressure Standard Deviation | Pressure variability indicator                                           |
| Optimization Time           | Recorded optimization execution time                                     |
| Convergence Rate            | Change in the optimization score across the recorded convergence history |

The application also reports the configured PSO parameters and the range of optimized PRV settings.

---

## PSO Convergence

The optimization workflow records the best objective function value during PSO iterations.

A convergence plot can be generated to visualize changes in the global best score.

Example output category:

```text
figures/
└── pso_convergence.png
```

A decreasing objective value indicates improvement according to the implemented penalty-based objective function.

The convergence plot should be interpreted within the scope of the current simplified optimization model.

## Reproducibility

Result values depend on:

* Input dataset
* Dataset preprocessing
* Missing-value patterns
* Outlier filtering
* Train/test split
* Hyperparameter search
* Model configuration
* PSO initialization
* Operational input values

The machine learning workflow uses a fixed random state where configured.

The PSO implementation currently uses NumPy-based random initialization without a dedicated optimization seed.

Therefore, optimization results may vary between executions.

---

## Data and Infrastructure Privacy

Real operational water network datasets may contain sensitive infrastructure information.

Raw operational datasets and infrastructure-specific identifiers should not be published unless explicit authorization has been obtained.

Public result files should use:

* Synthetic data
* Anonymized network identifiers
* Aggregated metrics
* Non-sensitive visualizations

---

## Interpretation Notice

The outputs generated by this project are intended for analytical experimentation, machine learning research, and engineering workflow development.

Optimization results produced by the current simplified pressure model require validation using a dedicated hydraulic simulation engine before operational use.

No result published in this repository should be interpreted as an operational control recommendation for a real water distribution system without independent engineering and hydraulic validation.
